# SenseArt

**Universal accessibility layer for gigapixel art viewers.**

SenseArt wraps [OpenSeadragon](https://openseadragon.github.io/) (OSD) and transforms it from a visual-only canvas into a fully navigable, screen-reader-friendly experience ‚Äî without modifying the underlying viewer.

---

## The Problem: Two Blind Spots in Art Digitization

### 1. The OpenSeadragon Canvas Trap

OpenSeadragon renders everything onto a `<canvas>` element. From an assistive technology perspective, canvas is a "black box": it has no semantic structure, no focusable children, and no textual content for screen readers to announce. OSD's keyboard navigation is rudimentary ‚Äî it moves the viewport but never communicates *where* the user is inside the image or *what* is visually significant.

Referenced upstream issues: [openseadragon#1478](https://github.com/openseadragon/openseadragon/issues/1478), [openseadragon#628](https://github.com/openseadragon/openseadragon/issues/628).

### 2. The Storiiies Linearity Problem

[Storiiies by Cogapp](https://storiiies.cogapp.com/) solved annotation-driven tours beautifully ‚Äî but only for sighted users following a curator-defined path. A blind user has no *agency*: they can only listen to the sequence chosen for them. There is no way to say "I want to explore the top-left corner" or "take me to the highest-detail area".

---

## The SenseArt Solution

SenseArt restores **agency** through three layered mechanisms:

| Layer | Mechanism | Technology | Status |
|---|---|---|---|
| Navigation | Semantic Grid Overlay | WAI-ARIA 1.2, Focus Management | Phase 1 ‚Äî in progress |
| Perception | Color Sonification | Tone.js | Phase 2 ‚Äî planned |
| Understanding | AI Mental Map | Multimodal AI (pluggable) | Phase 3 ‚Äî planned |

### Semantic Grid Overlay

A transparent `N√óN` grid of ARIA `gridcell` buttons floats above the canvas. Each cell maps to a spatial region of the artwork. Arrow keys / Tab move the focus highlight between cells without changing the viewport. Pressing `Enter` or `Space` zooms OSD into the focused region (relative to the viewport visible at activation time). The current position is always announced via `aria-live`.

### Color Sonification

Pixels under the focused cell are sampled. Luminosity maps to pitch, saturation maps to a low-pass filter cutoff. A blind user can "hear" the transition from a dark shadow to a bright highlight in real time.

### AI Mental Map (Phase 3)

Each grid cell receives a semantic label and a set of "sensory tags" (e.g., `["caldo", "testurizzato", "volto", "alto-contrasto"]`) generated by a multimodal AI model. The AI layer is **pluggable**: OpenAI, open-weight models via HuggingFace Inference API, or local Ollama instances all implement the same `ArtworkMapProvider` interface.

## Quick Start

```typescript
import OpenSeadragon from 'openseadragon'
import { SenseArtViewer } from 'sense-art'

const viewer = OpenSeadragon({ id: 'osd-container', tileSources: '...' })

const senseArt = new SenseArtViewer(viewer, {
  grid: { rows: 3, columns: 3 },
  sonification: { enabled: false }, // Phase 2
})

senseArt.mount()   // inject overlay into DOM
senseArt.enable()  // or press Alt+A to toggle
```

> `Alt+A` toggles the accessibility layer on/off at any time. On macOS with non-US keyboards (e.g. Italian), `Option+A` produces a composed character (`√•`); SenseArt uses `e.code` as fallback so the shortcut works regardless of keyboard layout.

## Keyboard Shortcuts

| Key | Action | Status |
|---|---|---|
| `Alt+A` | Toggle SenseArt accessibility layer | Implemented |
| `Arrow keys` | Move between grid cells (with wrap-around) | Implemented |
| `Tab` / `Shift+Tab` | Cycle linearly through all cells | Implemented |
| `Escape` | Reset zoom to full image, return to cell (0,0) | Implemented |
| `Enter` / `Space` | Zoom into focused cell (explicit intent) | Implemented |

## API

### `SenseArtViewer`

```typescript
const senseArt = new SenseArtViewer(viewer: OpenSeadragon.Viewer, options?: SenseArtOptions)

senseArt.mount()    // inject overlay, register Alt+A listener
senseArt.unmount()  // full cleanup ‚Äî removes overlay and all listeners
senseArt.enable()   // activate keyboard interception + announce
senseArt.disable()  // deactivate without removing overlay
senseArt.toggle()   // called internally by Alt+A
```

### `SenseArtOptions`

```typescript
interface SenseArtOptions {
  grid?: {
    rows: number     // default: 3
    columns: number  // default: 3
  }
  sonification?: {
    enabled: boolean
    toneDurationMs?: number  // default: 800ms
  }
  ai?: {
    provider?: 'mock' | 'gemini' | 'openai' | 'huggingface' | 'ollama'  // default: 'gemini'
    apiKey?: string    // required for 'gemini' and 'openai'
    model?: string     // overrides per-provider default (e.g. 'gemini-1.5-pro')
    baseUrl?: string   // custom endpoint (Ollama, Vertex AI, etc.)
    imageUrl?: string  // fallback when canvas is unavailable
  }
}
```

### AI loading events

`SenseArtViewer` dispatches two custom events on the OSD container during each AI hydration. Use them to show a loading indicator in your UI:

```typescript
osdEl.addEventListener('senseArt:ai-loading', () => { /* show spinner */ })
osdEl.addEventListener('senseArt:ai-ready',   () => { /* hide spinner */ })
```

`senseArt:ai-ready` fires in the `finally` block ‚Äî always emitted even on provider error, so the spinner never gets stuck. See [AI_INTEGRATION.md](./AI_INTEGRATION.md) for the full integration guide.

## Conventional Commits & Release

This repo enforces [Conventional Commits](https://www.conventionalcommits.org/):

```
feat(sense-art): add ResizeObserver for overlay sync
fix(sense-art): correct role="row" nesting in A11yOverlay
docs(docs): update ARCHITECTURE.md with data flow diagram
```

**Scopes**: `sense-art` ¬∑ `demo-osd` ¬∑ `demo-storiiies` ¬∑ `ci` ¬∑ `docs` ¬∑ `deps` ¬∑ `release`

Releases are **triggered manually** via GitHub Actions ‚Üí **Release ‚Üí Run workflow**. `semantic-release` analyzes commits since the last tag and determines the version bump:
- `fix:` ‚Üí patch (0.1.0 ‚Üí 0.1.1)
- `feat:` ‚Üí minor (0.1.0 ‚Üí 0.2.0)
- `feat!:` or `BREAKING CHANGE:` ‚Üí major (0.1.0 ‚Üí 1.0.0)

Each release automatically updates `CHANGELOG.md` and creates a GitHub Release with generated notes.

### npm Trusted Publishing (OIDC)

Publishing uses **npm Trusted Publishing** ‚Äî no `NPM_TOKEN` secret required. GitHub Actions authenticates with npm via a short-lived OIDC identity token (expires after each run). Every published version includes a **provenance attestation** verifiable with `npm audit signatures sense-art`.

**One-time setup on npmjs.com** (required before the first release):
1. Profile ‚Üí `sense-art` package ‚Üí **Settings ‚Üí Publishing ‚Üí Trusted Publishers**
2. **Add a publisher** ‚Üí GitHub
3. Owner: `micheleabbrescia` ¬∑ Repository: `sense-art` ¬∑ Workflow: `release.yml`

No secrets to add on GitHub ‚Äî `GITHUB_TOKEN` (injected automatically by Actions) is sufficient.
